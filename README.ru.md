# Vidlatte

[English](README.md) | [中文](README.zh.md)

Автономное приложение для генерации изображений с помощью ИИ на базе ComfyUI. Подключайтесь к одному или нескольким серверам ComfyUI, генерируйте изображения с полным контролем над моделями, LoRA, ControlNet и другими параметрами — через чистый кроссплатформенный интерфейс на Flutter.

## Возможности

- **Поддержка нескольких серверов** — подключение к нескольким экземплярам ComfyUI, переключение между ними на лету
- **Текст в изображение и изображение в изображение** — полные рабочие процессы txt2img и img2img с загрузкой эталонных изображений
- **Inpainting** — масочное инпейнтинг с настройкой размера кисти и силы изменения
- **ControlNet** — применение моделей ControlNet с контрольными изображениями и настройкой силы
- **Управление LoRA** — выбор нескольких LoRA с индивидуальным контролем веса, автоматическое получение триггерных слов, скрытие неиспользуемых LoRA
- **Просмотр моделей** — просмотр доступных чекпойнтов, LoRA и моделей ControlNet с каждого сервера
- **Очередь генерации** — постановка нескольких задач в очередь, отмена или повтор отдельных задач
- **Автоматический режим** — автоматическая пакетная генерация с созданием промптов через LLM
- **Интеграция LLM** — подключение к LLM-серверам, совместимым с OpenAI (LM Studio и др.) для помощи с промптами
- **Галерея** — галерея с поиском и фильтрацией, коллекции, избранное, скрытые изображения и защита паролем
- **Студия** — организация сессий генерации с сохранением истории по каждой сессии
- **История промптов** — быстрый доступ к предыдущим промптам с повторным использованием в один клик
- **Hires-fix** — опциональный проход высокоразрешающего исправления для сгенерированных изображений
- **Адаптивный интерфейс** — автоматическая адаптация под телефон, планшет и десктоп
- **Тёмная / Светлая / Системная тема** — следование системной теме или ручное переключение
- **Офлайн-хранение** — все данные хранятся локально через Hive, без облачной зависимости

## Установка

### Ночные сборки

Предварительно собранные бинарные файлы публикуются автоматически при каждом коммите через GitHub Actions. Скачайте последнюю версию с [nightly release](https://github.com/openlyst/vidlatte-flutter/releases/tag/nightly).

### Сборка из исходного кода

```bash
git clone https://github.com/openlyst/vidlatte-flutter.git
cd vidlatte-flutter
flutter pub get
flutter run
```

Сборка release-артефакта для конкретной платформы:

```bash
flutter build apk --release      # Android
flutter build ios --release      # iOS
flutter build macos --release    # macOS
flutter build linux --release    # Linux
flutter build windows --release  # Windows
flutter build web --release      # Web
```

## Использование

1. Откройте приложение и перейдите в **Настройки**
2. Добавьте сервер ComfyUI (URL, опциональные заголовки аутентификации)
3. Нажмите проверку здоровья для подтверждения подключения
4. Перейдите в **Создание** и выберите модель из выпадающего списка
5. Введите промпт, настройте параметры (креативность, шаги, размеры, LoRA) по необходимости
6. Нажмите **Сгенерировать** — изображения появятся в панели результатов и сохранятся в галерее
7. (Опционально) Включите **Автоматический режим** для пакетной генерации с помощью LLM

Для img2img или inpainting переключите режим в панели создания и предоставьте эталонное изображение.

## Архитектура

Vidlatte использует архитектуру на основе функций с BLoC для управления состоянием:

```
lib/
├── app.dart                  # Точка входа приложения с настройкой BlocProvider
├── bloc/                     # Слой BLoC (настройки, серверы, генерация, галерея, студия, LLM, автогенерация, история промптов)
├── config/                   # Тема, константы, конфигурация приложения
├── data/                     # Модели и слой данных
├── i18n/                     # Локализация (en, zh, ru)
├── presentation/             # Слой UI
│   ├── navigation/           # Конфигурация GoRouter
│   ├── pages/                # Создание, Галерея, Студия, Настройки, Просмотр, Inpainting
│   └── widgets/              # Переиспользуемые UI-компоненты
└── services/                 # Сервис ComfyUI, сервис LLM, сервис хранилища
```

**Ключевые технологии:**

- **Управление состоянием:** flutter_bloc
- **Навигация:** go_router
- **Сеть:** dio, web_socket_channel
- **Хранение:** Hive, shared_preferences, path_provider
- **UI:** Material 3, cached_network_image, photo_view, shimmer, flutter_staggered_grid_view

## Ночные сборки

Рабочий процесс GitHub Actions запускается при каждом пуше в любую ветку. Он параллельно собирает все шесть платформ и публикует GitHub Release с тегом `nightly` со всеми артефактами. Релиз автоматически заменяется при каждом запуске, поэтому всегда отражает последний коммит.

Если сборки для некоторых платформ завершаются с ошибкой, релиз всё равно публикуется, если хотя бы одна сборка прошла успешно.
